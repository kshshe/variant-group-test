# PDFNet

Базовый монорепозиторий для тестового задания Variant Group.

## Что внутри

- `server/` — Express API c endpoint `POST /api/generate`, защитой от DoS/DDoS на прикладном уровне, JSDoc и Swagger UI.
- `client/` — Vite + React + TypeScript клиент с `wouter`, `react-toastify`, `useLocalStorageState` и `DocumentsStorage`.

## Быстрый старт

```bash
npm install
cp server/.env.example server/.env
npm run dev
```

- Client: `http://localhost:5173`
- Swagger UI: `http://localhost:3001/api/docs`

## Команды

```bash
npm run dev
npm run build
npm run test
```

# Workflow

1. Инициализация проекта

Я получил тестовое, когда был не дома. После того, как ознакомился с требованиями, написал телеграм-боту, который запущен у меня локально, запрос, чтобы он создал директорию для проекта и экспортировал в неё требования.

Дальше был примерно такой запрос: "Сделай там базовую структуру — простой сервер на express js с одним эндпоинтом — generate, который принимает поля из формы + обработку для защиты от ddos и dos атак. Добавь jsdoc и swagger ui. Используй пакет openai, промпт вынеси в отдельный файл. Клиент - vite, react. Там же заведи типы для api-запросов (нативный fetch). Заведи класс DocumentsStorage, с абсолютными методами add, delete, статическим приватным полем storageProvider, где по умолчанию будет лежать window.localStorage, но типизация должна позволять менять провайдера. Заведи компонент отображения списка с карточками и страницу создания. Без дизайна. Используй wouter. Стейт формы держи в хуках useLocalStorageState. При отправке ставь useState loading в true. Обработку до трех ретраев делай тихо на уровне api-wrapper. В случае ошибки выводи toast ошибки через react-toastify. Инициализируй git и gitignore. Скопируй .env из /Volumes/Projects/Personal/BookWriter, там есть openai ключ. Запусти проект, открой его в браузере, создай пару текстов, пришли скриншоты"

Когда вернулся, получил работающий прототип, требующий разработки UI с нуля и доработок по всему остальному.

2. Дизайн-система

Вынес иконки, инпуты, тексты и кнопки со всеми размерами и цветами из макета. Для отступов четкой закономерности на этом этапе не увидел, поэтому отложил формирование spacing-системы на более поздние этапы, когда затрону большую часть стилей в макетах.
