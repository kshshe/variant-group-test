# PDFNet

[![Website](https://img.shields.io/website?url=https%3A%2F%2Fvariant.alekseilaud.dev&label=variant.alekseilaud.dev)](https://variant.alekseilaud.dev)

[Swagger UI](https://variant.alekseilaud.dev/api/docs/)

[Design System](https://variant.alekseilaud.dev/design/)

Базовый монорепозиторий для тестового задания Variant Group.

## Что внутри

- `server/` — Express API c endpoint `POST /api/generate`, защитой от DoS/DDoS на прикладном уровне, JSDoc и Swagger UI.
- `client/` — Vite + React + TypeScript клиент с `wouter`, `react-toastify` и `DocumentsStorage`.

## Быстрый старт

```bash
npm install
cp server/.env.example server/.env
npm run dev
```

- Client: `http://localhost:5173`
- Swagger UI: `http://localhost:3001/api/docs`

## Команды

```bash
npm run dev
npm run build
npm run test
```

# Workflow

1. Инициализация проекта

Я получил тестовое, когда был не дома. После того, как ознакомился с требованиями, написал телеграм-боту, который запущен у меня локально, запрос, чтобы он создал директорию для проекта и экспортировал в неё требования.

Дальше запросил реализацию базовой структуры — простой сервер на express js с одним эндпоинтом — generate, который принимает поля из формы + обработку для защиты от ddos и dos атак. JSDoc и Swagger. Клиент - vite, react. Там же типы для api-запросов (нативный fetch). Описал класс DocumentsStorage с абсолютными методами add, delete, статическим приватным полем storageProvider, где по умолчанию будет лежать window.localStorage, но типизация должна была позволять менять провайдера. Подключил wouter (react-router посчитал перебором для двух страниц). Добавил обработку трех ретраев на уровне api-wrapper. Для ошибок подключил react-toastify.

Когда вернулся, получил работающий прототип, требующий разработки UI с нуля и доработок по всему остальному. Начиная со следующего этапа большую часть работы делал вручную. Пока что (в течение последних лет работы с ИИ-инструментами) я ни разу не смог заставить LLM нормально воспринимать макеты так, чтобы за ними не приходилось переделывать 90% кода, поэтому предпочитаю такие вещи делать в более ручном режиме.

2. Дизайн-система

Вынес иконки, инпуты, тексты и кнопки со всеми размерами и цветами из макета. Для отступов четкой закономерности на этом этапе не увидел, поэтому отложил формирование spacing-системы на более поздние этапы, когда затрону большую часть стилей в макетах. Сразу завел Ladle для наглядности (аналог Storybook). В результате по spacing-системе к какому-то вердикту так и не пришел. Чувствуется, что значения не рандомные (часто 12, 16, 24, 32 пикселя), но связывающего их паттерна я не увидел.

3. Страницы

Сделал обе страницы, дополнительно выделив утилитарные компоненты - Container, Page, Progress. Для формы вынес валидацию в общий с бэкендом пакет. В остальном страницы не потребовали доработок, дизайн-система позволила сделать большую часть без кастомизации.

4. Лоадер

Т.к. в фигме к таймингу анимации у меня не было доступа, я сделал его, ориентируясь на прототип.

5. Мобильная версия

Сделал базовый вариант - один брейкпоинт при сужении viewport просто уменьшает контент, второй, при достижении мобильных размеров, перестраивает сетки и flex-контейнеры в столбец. Для обоих брейкпоинтов все шрифты и отступы скейлятся в общей утилите.
