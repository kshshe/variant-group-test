# PDFNet — требования по тестовому заданию Variant Group

Источник: https://variantnet.notion.site/React-Developer-Test-Assignment-Variant-Group-d7a1e3460dc643958eb57a0518ce84b2

## Контекст
Нужно разработать небольшое HR-tech приложение для генерации текстов сопроводительных писем для соискателей.

## Основная задача
Приложение должно состоять из двух экранов:
1. Дашборд со списком созданных писем
2. Форма для генерации нового письма

## Функциональные требования

### 1. Генерация писем
- Пользователь должен иметь возможность создать новое сопроводительное письмо через форму.
- Генерация текста письма должна происходить с использованием AI.
- Нужно подключиться к API OpenAI или любому другому LLM API на выбор.
- Нужно продумать prompt и связать его с данными формы, чтобы модель генерировала динамические тексты.

### 2. Дашборд
- На дашборде должен отображаться список созданных писем.
- Нужно самостоятельно спроектировать empty state для дашборда.

### 3. Мотивационный баннер
- Цель приложения — мотивировать пользователя создать хотя бы 5 писем.
- Пока писем меньше 5, должен отображаться баннер с текущим числом созданных писем.

### 4. Сохранение данных в браузере
- Приложение должно восстанавливать письма, созданные в этом же браузере.
- После закрытия и повторного открытия вкладки список писем должен сохраняться.
- Для этого нужно использовать browser API (локальное хранение данных в браузере).

### 5. Адаптивность
- В исходных макетах нет мобильной версии.
- Нужно самостоятельно спроектировать и реализовать мобильную версию.

## Технические требования
- Обязательно использовать React.
- Обязательно использовать TypeScript.
- Желательно не использовать Tailwind.
- Остальной стек — на усмотрение разработчика.

## Дизайн и UI
- В макетах специально не используется готовая дизайн-система или React UI kit.
- Одна из ключевых частей задания — самостоятельно выявить закономерности интерфейса и систематизировать компоненты и стили.
- При реализации важны качество вёрстки, консистентность UI и продуманность собственной мини дизайн-системы.

## Практический вывод для проекта PDFNet
Минимальный MVP должен включать:
- экран списка писем;
- экран создания письма;
- интеграцию с LLM для генерации текста;
- локальное сохранение писем в браузере;
- баннер прогресса до 5 писем;
- empty state;
- мобильную адаптацию.

## Что стоит предусмотреть сразу
- Типизированную доменную модель письма
- Слои: UI / state / services / storage / ai
- Переиспользуемые UI-компоненты вместо ad-hoc стилей
- Обработку ошибок генерации
- Loading / disabled / retry состояния
- Валидирование формы
- Возможность быстро задеплоить проект
