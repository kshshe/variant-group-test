Задача
Разработать небольшое HR-tech приложение для генерации текстов сопроводительных писем (пользователи – те кто ищет работу), состоящее из двух экранов:

- Дашборд со списком созданных писем
- Форма для генерации нового письма

Детали

- Задача приложения — замотивировать пользователя создать хотя бы 5 писем. До тех пор пока писем меньше, отображается баннер с текущим числом.
- Приложение должно уметь восстанавливать сопроводительные письма, которые были созданы в этом же браузере (за счет API браузера). Если закрыть вкладку и открыть снова — список созданных писем должен быть таким же.
- В макетах специально не используется никакой дизайн-системы или готового React UI kit-а. Поиск закономерностей, систематизация компонентов и стилей — одна из ключевых частей тестового задания.
- Спроектируйте empty state дашборда на свой вкус.
- В макетах отсутствует мобильная версия — вам нужно создать ее самостоятельно.
- Нужно внедрить AI для написания писем. Подключиться к API OpenAI (или любому другому на ваше усмотрение), создать prompt и связать его с данными формы, чтобы модель писала динамические тексты.
- Если напишите на vlad@variant.net, мы пришлем вам временный API ключ для работы с ChatGPT.

Стек

- Обязательно использовать React и TypeScript.
- Желательно не использовать Tailwind.
- Остальное — на ваше усмотрение.

Про AI-инструменты
Использование Cursor, Claude Code, Codex, Copilot — да чего угодно — поощряется. Мы сами пишем код и создаем дизайн в связке с AI каждый день и ждём того же от специалистов, которых нанимаем.

Будем рады, если в README коротко расскажете про свой workflow: какими инструментами пользовались, где они помогли, где пришлось спорить с моделью и дописывать руками, какой кусок кода считаете самым «своим». Это не для проверки — нам интересно, как устроена ваша работа.
